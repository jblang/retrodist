# retrolib

This directory contains the host-side shell library sourced by the `retro`
entry point at startup. Every file here is sourced unconditionally, so all
functions are available regardless of the command being run.

## Documentation Map

- [Compatibility Notes](#compatibility-notes): portability rules for host-side code.
- [Shared Utilities](#shared-utilities): helpers and logging.
- [Config and Staging](#config-and-staging): config lookup and `qemu.d/`.
- [Media Commands](#media-commands): download and extraction helpers.
- [QEMU Runtime](#qemu-runtime): QEMU variables, profiles, drives, ports, and networking.
- [Automation Helpers](#automation-helpers): QMP and scripted install primitives.
- [Slackware Tagfiles](#slackware-tagfiles): Slackware package selection internals.

## Compatibility Notes

The host-side library should run on modern GNU and BSD/macOS userlands.

Use Bash syntax supported by GNU bash 3.2, the version Apple ships on macOS.
Do not use Bash 4+ features such as associative arrays, `mapfile`, `readarray`,
`wait -n`, or `|&`.

External command use should stay within the option subsets available in both
BSD and GNU variants on supported hosts. When adding or changing command-line
flags, check behavior on both BSD and GNU userlands instead of assuming GNU-only
extensions are available everywhere.

## Shared Utilities

### `helpers.sh`

Shared utilities used across all other modules.

- `retro_config_file [DIR] FILE`
  Searches `DIR` (default: `$CONFDIR`) then its parent for `FILE`. Prints the
  path and returns 0 when found; returns 1 when not found. Config files can be
  shared at the version level by placing them one directory above the variant.

- `retro_prereq`
  Detects the host package manager (Homebrew, apt-get, dnf, pacman, MSYS2
  pacman) and installs the required tools for that platform, including QEMU,
  archive/download tools, CD image tools, port-inspection tools, JSON helpers,
  mtools, and SSH client tools where they are not provided by the base system.
  Supports `--dry-run`.

- `autoinst_prep`
  Stages the `autoinst/` tree onto `qemu.d/fat/` for the current config. Copies
  `autoinst.sh` as `fat/autoinst`, copies the whole tree to `fat/autoinst.d/`,
  then copies the distro's `autoinst.sh` and `autoconf.sh` manifests to
  `fat/autoinst.d/distro/`. See [autoinst/README.md](../autoinst/README.md).

### `logging.sh`

Host-side logging helpers. Messages are written to stderr and, when
`RETRO_LOG_FILE` is set, appended to that file as well.

- `log_debug MESSAGE...`
  Logs only when `RETRO_DEBUG=1`.

- `log_info MESSAGE...`
  Logs an informational message.

- `log_warn MESSAGE...`
  Logs a warning.

- `log_error MESSAGE...`
  Logs an error.

- `log_attention MESSAGE...`
  Logs an attention-grabbing status message.

- `log_div`
  Logs a divider line.

- `die MESSAGE...`
  Logs an error and exits with status 1.

Each level uses a configurable marker variable. The defaults are emoji, but the
values can be plain text, ANSI-styled text, or an empty string:

```bash
RETRO_LOG_INFO=INFO retro boot slackware/3.0/walnut
RETRO_LOG_WARN=$'\e[1;33mWARN\e[0m' retro boot slackware/3.0/walnut
RETRO_LOG_ERROR= retro boot slackware/3.0/walnut
```

Marker variables:

- `RETRO_LOG_DEBUG`
- `RETRO_LOG_INFO`
- `RETRO_LOG_WARN`
- `RETRO_LOG_ERROR`
- `RETRO_LOG_ATTENTION`
- `RETRO_LOG_DIVIDER`

## Config and Staging

### Config Directory Hierarchy

Distro configs live at `distro/version/variant/` (e.g., `slackware/3.0/walnut/`).
`retro_config_file NAME` searches `$CONFDIR` (the variant directory) first, then
its parent, so files can be shared at the version level.

All per-config files are optional. See [CONTRIBUTING.md](../CONTRIBUTING.md)
for the usual flow when adding a distro.

| File | Purpose |
|---|---|
| `config.sh` | Distro-wide settings; loaded before QEMU config |
| `qemu.sh` | Set `QEMU_PROFILE`, RAM, disk size, net device, extra QEMU args |
| `extract.sh` | Set `EXTRACT_*` vars and call `extract_install_files` |
| `ks.cfg` | Red Hat Kickstart file copied to the staged boot floppy image |
| `script.sh` | Host-side install automation; sourced after QEMU starts |
| `autoinst.sh` | In-guest install manifest; copied to `fat/autoinst.d/distro/` |
| `autoconf.sh` | In-guest first-boot manifest; copied to `fat/autoinst.d/distro/` |
| `download.txt` | `filename url` pairs for wget |
| `slackmirror.txt` | Slackware version for official mirror download |
| `debmirror.txt` | Debian release name for archive.debian.org download |
| `download.sh` | Custom download logic |
| `cdrom.txt` | Reference to a `cdrom/` config; ISOs are linked into `qemu.d/` |
| `*.tag` | Slackware tagset: package-selection rules (`series package state`). `default`/`full`/`base` are conventional; see [`slackware.sh`](#slackwaresh) |

### `qemu.d/` — Staging Area

Generated by `retro extract`, `retro boot`, `retro install`, or `retro package`
for each distro config. Contains:

- `hda.img` — primary hard disk, created when startup media is present
- `boot.img` → symlink to the boot floppy image
- `root.img` → symlink to the root floppy image
- `install.iso` → symlink to the install ISO
- `fda.img`, `hdc.iso`, etc. — optional additional media
- `fat/` — when present, exposed to the guest as a writable FAT disk at `/dev/hdb1`
- `fat/autoinst` — in-guest install runner, staged for automated installs
- `fat/autoinst.d/` — shared runtime and distro manifests, staged for automated installs (see [autoinst/README.md](../autoinst/README.md))
- `qmp.sock` — QMP Unix socket (live while QEMU is running)
- `ttyS0.sock`…`ttyS3.sock`, `lp0.sock` — serial and parallel port sockets

Do not edit files under `qemu.d/fat/autoinst.d/` directly. When present, they
are generated copies of the shared runtime and distro manifests. Edit files under
`autoinst/` or the distro config instead.

## Media Commands

### `download.sh`

Asset download helpers and top-level `retro download` handler.

- `download_list MANIFEST DESTDIR`
  Downloads `filename url` pairs from a manifest file using wget, skipping
  files that already exist.

- `download_files URL_BASE DESTDIR FILE…`
  Downloads named files from a base URL.

- `download_directories URL_BASE DESTDIR DIR…`
  Recursively downloads named directories from a base URL using wget
  `--recursive`, computing `--cut-dirs` automatically.

- `download_slackware VERSION`
  Downloads a full Slackware release tree from mirrors.slackware.com.

- `download_debian RELEASE`
  Downloads a Debian release from archive.debian.org. Handles per-release
  layout differences (Debian 0.93R6 through post-potato).

- `download_config_assets CONFIG_DIR DESTDIR`
  Dispatches all configured download mechanisms for a config directory:
  `download.txt`, `slackmirror.txt`, `debmirror.txt`, and `download.sh`.

- `download_cdrom_assets CONFIG_DIR` / `link_cdrom_isos CONFIG_DIR DESTDIR`
  Downloads assets for a `cdrom/` config referenced by `cdrom.txt` and links
  resulting ISO files into the distro extraction directory.

- `retro_download`
  Top-level handler for `retro download CONFIG`. Calls `download_all` against
  `$CONFDIR` and `$ORIGDIR`.

### `extract.sh`

Media extraction and staging helpers and the top-level `retro extract` handler.

Extraction is controlled by setting `EXTRACT_*` variables before calling
`extract_install_files`. The function detects whether the source is an archive
or a directory and dispatches accordingly.

`EXTRACT_*` variables:

- `EXTRACT_SOURCE` — archive file or source directory. Relative paths are
  resolved under `$ORIGDIR`.
- `EXTRACT_BOOT_IMAGE` — boot floppy image name to extract.
- `EXTRACT_ROOT_IMAGE` — root floppy image name to extract.
- `EXTRACT_EXTRA_IMAGES` — array of additional image names to extract.
- `EXTRACT_FAT_FILES` — array of files to extract into `fat/`.
- `EXTRACT_PACKAGES` — directory path inside the archive to extract as
  `fat/packages/`. Use `"."` to extract the whole archive.

Key functions:

- `extract_install_files`
  Extracts or copies the configured images and packages, links `boot.img` and
  `root.img`, makes staged artifacts writable by the current user, and resets
  `EXTRACT_*` afterwards.

- `debian_extract_fat_image IMAGE DEST FILE…`
  Extracts selected files from a FAT image using 7z and lowercases filenames.

- `retro_link_boot_root BOOT ROOT`
  Updates the `boot.img` and `root.img` symlinks in `qemu.d/`.

- `retro_extract`
  Top-level handler for `retro extract CONFIG`. Runs `retro_download`, sources
  the distro's `extract.sh`, copies a configured Red Hat `ks.cfg` into
  `boot.img`, stages the autoinstall runtime and any configured distro
  autoinstall manifests, and writes `qemu.d/.extracted`. Later runs reuse the
  extracted files.

## QEMU Runtime

### `qemu.sh`

QEMU configuration, port allocation, drive building, and top-level boot,
install, reset, and package handlers.

#### Configuration variables

All `QEMU_*` variables can be set in the distro's `qemu.sh` or via the
environment. `load_qemu_config` applies defaults, sources the distro's
`qemu.sh`, resolves the profile, assigns ports, and finalizes network config.

System:

- `QEMU_SYSTEM` — QEMU binary. Default: `qemu-system-i386`.
- `QEMU_PROFILE` — era profile name. Default: `default`. See [Profiles](#profiles).
- `QEMU_MACHINE` — `-machine` argument. Set by profile.
- `QEMU_RAM` — RAM size. Set by profile.
- `QEMU_SMP` — CPU count. Default: `1`.
- `QEMU_ACCEL` — acceleration flags. Default: `-accel tcg`.
- `QEMU_EXTRA` — extra QEMU arguments appended verbatim.
- `QEMU_DISPLAY` — display backend flags. Default: `-display cocoa` on macOS,
  `-display gtk` elsewhere. Cocoa zoom/scaling options are added
  automatically only when running QEMU 11.x.

Storage:

- `QEMU_HD_FORMAT` — disk image format. Default: `qcow2`.
- `QEMU_HD_SIZE` — primary disk size. Set by profile.
- `QEMU_HD_CREATE_OPTIONS` — extra `-o` options for `qemu-img create`.
- `QEMU_HDA_OPTIONS` — extra options appended to the `hda` drive string.
- `QEMU_FDTYPE_A` / `QEMU_FDTYPE_B` — floppy type globals. Default: `144`.
- `QEMU_BOOT_ORDER` — `-boot order=` argument. Auto-set at install time.

Network:

- `QEMU_NET_TYPE` — `user` (default), `jump`, or `none`. See [Network modes](#network-modes).
- `QEMU_NET_DEVICE` — NIC model. Set by profile.
- `QEMU_INTERNET` — raw `-netdev`/`-device` string for internet access.
  Overrides `QEMU_NET_TYPE` when set.
- `QEMU_RETRONET` — raw `-netdev`/`-device` string for the `jump` network.
  Overrides `QEMU_NET_TYPE` when set.

Ports:

- `QEMU_SSH_BASE` / `QEMU_SSH_PORT` — host port for guest :22. Default base: 2200.
- `QEMU_TELNET_BASE` / `QEMU_TELNET_PORT` — host port for guest :23. Default base: 2300.
- `QEMU_HTTP_BASE` / `QEMU_HTTP_PORT` — host port for guest :80. Default base: 8000.
- `QEMU_MONITOR_BASE` / `QEMU_MONITOR_PORT` — telnet monitor port. Default base: 5555.
- `QEMU_QMP_SOCKET` — QMP Unix socket name. Default: `qmp.sock`. Set `none` to disable.

Serial / parallel:

- `QEMU_SERIAL_SOCKET_COUNT` — number of serial sockets. Default: `4`.
- `QEMU_SERIAL_SOCKET_PREFIX` — socket name prefix. Default: `ttyS`.
- `QEMU_PARALLEL_SOCKET_COUNT` — number of parallel sockets. Default: `1`.
- `QEMU_PARALLEL_SOCKET_PREFIX` — socket name prefix. Default: `lp`.

Install scripting:

- `QEMU_INSTALL_SCRIPT` — path to a `script.sh` sourced after QEMU starts.
  Auto-set from the distro's `script.sh` when running `retro install`.

#### Displays

By default, `retro` starts QEMU with a native window display backend: `gtk` on
Linux and `cocoa` on macOS. Override this by setting `QEMU_DISPLAY`, for
example:

```bash
QEMU_DISPLAY="-display sdl" retro boot slackware/3.0/walnut
```

If QEMU reports that the selected backend is unavailable, install a QEMU UI
backend package such as `qemu-ui-gtk` or choose another backend supported by
`qemu-system-i386 -display help`.

On macOS, Cocoa zoom-to-fit and zoom interpolation are enabled automatically
only when the selected backend is `cocoa` and the detected QEMU major version is
11. They are not enabled for QEMU 10.x or other display backends.

#### Profiles

Set `QEMU_PROFILE` in the distro's `qemu.sh` to select era-appropriate
hardware defaults. Any variable set before `qemu_apply_profile` is not
overwritten, so individual overrides still take effect.

| Profile | Machine | RAM | Disk | NIC |
|---|---|---|---|---|
| `default` | isapc | 16M | 500M | ne2k_isa |
| `linux-0.99` | isapc | 64M | 500M | ne2k_isa |
| `linux-1.0` | isapc | 64M | 512M | ne2k_isa |
| `linux-1.2` | isapc | 64M | 2G | ne2k_isa |
| `linux-2.0-isa` | isapc | 64M | 2G | ne2k_isa |
| `linux-2.0` | pc | 64M | 8G | tulip |
| `linux-2.2` | pc | 64M | 8G | tulip |
| `linux-2.4` | pc | 128M | 8G | tulip |

#### Network modes

Set `QEMU_NET_TYPE` in the distro's `qemu.sh` or in the environment.

- `user` (default) — SLIRP NAT with host port forwards: `QEMU_SSH_PORT`→:22,
  `QEMU_TELNET_PORT`→:23, `QEMU_HTTP_PORT`→:80.
- `jump` — socket network mode documented in [JUMP.md](../JUMP.md).
- `none` — no network interface.

Setting `QEMU_INTERNET` or `QEMU_RETRONET` directly bypasses `QEMU_NET_TYPE`.

Examples:

```bash
retro boot slackware/3.0/walnut
QEMU_NET_TYPE=none retro boot slackware/3.0/walnut
```

#### Preparation Flow

`retro boot` and `retro install` both call `extract` first, then prepare QEMU
from the files in `qemu.d/`:

1. Load `config.sh` from the config or parent directory if present.
2. Set default `QEMU_*` values, source `qemu.sh`, apply `QEMU_PROFILE`, assign
   host ports, and finish network/display configuration.
3. Select command-specific install media. During `boot` and `install`,
   `boot.img` is attached to the first floppy drive when present, and
   `install.iso` is attached as the CD-ROM when no `hdc.iso` exists. During
   `install`, boot order prefers floppy or CD-ROM install media.
4. Create `hda.img` with `QEMU_HD_SIZE`, `QEMU_HD_FORMAT`, and optional
   `QEMU_HD_CREATE_OPTIONS` when it does not exist and startup media is present.
5. Attach guest disks from existing files and directories.
6. Create serial and parallel Unix socket chardevs, build the QEMU command, and
   print assigned endpoints, forwards, disks, character devices, and command.

Drive attachment rules:

- `fda.img` and `fdb.img` attach as floppy drives.
- `hda.img`, `hdb.img`, `hdc.img`, and `hdd.img` attach as IDE disks.
- `hda.iso`, `hdb.iso`, `hdc.iso`, and `hdd.iso` attach as IDE CD-ROMs.
- `fat/` attaches as a writable FAT disk on the second IDE drive when no `hdb`
  image or ISO exists. Guests usually mount it as `/dev/hdb1`.
- Directories named `fda`, `fdb`, `hda`, `hdb`, `hdc`, or `hdd` attach as
  writable FAT-backed drives.

#### Top-level handlers

- `retro_boot` — `retro boot CONFIG [QEMU_ARGS…]`
  Extracts, builds drives, assigns ports, starts QEMU. For `retro install`,
  also sources the install script via `qemu_run_with_install_script`.

- `retro_package` — `retro package CONFIG [--hda] [QEMU_ARGS…]`
  Calls `retro_boot`, writes `retro.sh` (POSIX sh) and `retro.bat` (Windows
  cmd) launchers into `qemu.d/`, and tars the directory.

- `retro_reset` — `retro reset CONFIG`
  Prompts and then removes `qemu.d/` and extracted files for the config.

### Host to VM Communication

QEMU exposes its monitor through a host loopback TCP port and QMP through a Unix
domain socket in `qemu.d/`. Guest serial and parallel ports are also forwarded
through Unix domain sockets. When user networking is enabled, guest TCP ports
are forwarded to host ports.

For each TCP listener, `retro` uses `lsof` to choose the lowest available port
from the 100 port numbers starting at the base port for that service. Each port
is incremented independently. The assigned ports and sockets are printed before
QEMU starts.

Typical first-VM endpoint assignment:

```text
QEMU endpoints:
  Monitor: 127.0.0.1:5555
  QMP:     /path/to/distro/qemu.d/qmp.sock

Guest ports:
  SSH:     127.0.0.1:2200 -> guest :22
  Telnet:  127.0.0.1:2300 -> guest :23
  HTTP:    127.0.0.1:8000 -> guest :80
```

Use the printed startup values rather than assuming a port number.

Monitor and QMP access:

- QEMU monitor: `telnet 127.0.0.1 5555`
- QMP protocol: `nc -U qemu.d/qmp.sock`

Guest TCP forwards:

- SSH: `ssh -p 2200 127.0.0.1`
- Telnet: `telnet 127.0.0.1 2300`
- HTTP: `http://127.0.0.1:8000/`

A forwarded port does not mean the guest has a service listening there. Telnet
is common in old distros; SSH and HTTP become common later.

Guest serial and parallel sockets:

- `/dev/ttyS0`: `qemu.d/ttyS0.sock`
- `/dev/ttyS1`: `qemu.d/ttyS1.sock`
- `/dev/ttyS2`: `qemu.d/ttyS2.sock`
- `/dev/ttyS3`: `qemu.d/ttyS3.sock`
- `/dev/lp0`: `qemu.d/lp0.sock`

Connect to a serial socket with `minicom -D 'unix#ttyS0.sock'`. Connect to the
parallel socket with `nc -U qemu.d/lp0.sock`.

#### Internet Access

With `QEMU_NET_TYPE=user`, the VM has outbound internet access through QEMU's
SLIRP NAT. Old distributions are riddled with security holes and should never
be placed directly on a public network.

Set `QEMU_NET_TYPE=none` to completely isolate a VM. This also disables port
forwarding.

Many old distros cannot use modern network protocols. Old browsers without
modern HTTPS ciphers can still browse through HTTP proxies such as
[FrogFind](http://frogfind.com/).

## Automation Helpers

### `qmp.sh`

QMP protocol helpers. Used by `script.sh` during automated installs and
exposed through the `qmp` entry point for interactive use.

#### QMP CLI

The top-level `qmp` helper connects to `qemu.d/qmp.sock` by default when run
from a distro config directory, or `qmp.sock` when run from inside `qemu.d/`.
Set `QEMU_QMP_SOCKET` or pass `qmp -s SOCKET` to use a different QMP socket.

- `qmp dump-screen`: dump the full VGA text memory range as text.
- `qmp dump-screen -n`: prefix rows with line numbers.
- `qmp send-key ret`: send a literal QEMU sendkey key sequence.
- `printf 'ramdisk' | qmp send-stdin`: type stdin into the VM.
- `qmp send-text -n 'ramdisk'`: type text into the VM and press ENTER.
- `qmp change-image root.img`: change the default floppy drive image.
- `qmp eject-disk`: eject the default floppy drive image.

Connection variables:

- `QEMU_QMP_SOCKET` — QMP socket name or path. Default: `qmp.sock`.
- `QMP_TIMEOUT` — `nc` timeout in seconds. Default: `1`.

VGA dump variables:

- `VGA_ADDR` — physical address of VGA text memory. Default: `0xb8000`.
- `VGA_COLS` / `VGA_ROWS` — screen dimensions. Default: 80×25.
- `VGA_MEM_BYTES` — bytes to read. Default: `32768`.

Key functions:

- `qmp_init` — sets defaults, checks prerequisites (nc, jq, xxd), validates VGA config.
- `qmp_hmp_command COMMAND` — sends an HMP command through QMP.
- `qmp_sendkey KEY` — sends a QEMU sendkey token.
- `qmp_send_string TEXT` — types a string into the guest character by character.
- `qmp_change_image IMAGE [DEVICE]` — changes floppy media at runtime.
- `qmp_eject_disk [DEVICE]` — ejects media from a block device.
- `qmp_boot_disk DEVICE` — sets the next boot device order.
- `qmp_vga_dump_text` — dumps and decodes VGA text memory as a plain-text screen.

### `script.sh`

QMP-driven install automation primitives. These are sourced when QEMU starts
during `retro install` and are the building blocks for per-distro `script.sh`
manifests. A script waits for the screen to reach a known state, then sends a
key, a line of text, or swaps media. A per-distro `script.sh` is written by
composing these primitives directly. Prefer [`serial_shell`](#serialsh) for
shell commands whose output does not need to stay on the VGA screen.

`SCRIPT_AUTOINST_COMMAND` holds the shell one-liner that mounts the staged FAT
media at `/retro` and runs `autoinst`; send it with
`serial_shell --no-wait` once a shell prompt appears.

`SCRIPT_AUTOCONF_COMMAND` holds the first-boot shell one-liner that mounts the
staged FAT media at `/retro` if needed, then runs `autoinst.d/autoconf.sh`.
`script_run_autoconf [PASSWORD]` logs in as root after first boot and sends
that command.

Timing variables:

- `WAIT_INTERVAL` — polling interval in seconds. Default: `1`. Wait helpers do
  not time out; use a QEMU-level stop or interrupt if an install stalls.

Screen, keyboard, and serial APIs are intentionally separate. `screen_*`
helpers scrape VGA text memory over QMP and never read the serial pipe.
`kb_*` helpers type through QMP keyboard input and never write to the serial
pipe. Serial helpers live in [`serial.sh`](#serialsh).

Functions:

- `screen_wait [-l] TEXT [TEXT ...]`
  Polls VGA text memory until each `TEXT` appears anywhere on screen, in the
  argument order. Pass `-l` to match trimmed full lines instead.

- `kb_press_key KEY [COUNT]`
  Sends one QEMU sendkey token (e.g. `ret`, `spc`, `ctrl-alt-delete`), repeated
  `COUNT` times when provided.

- `kb_send_line TEXT`
  Types `TEXT` into the guest and presses Return.

- `script_run_autoconf [PASSWORD]`
  Waits for a first-boot login prompt, logs in as root, and runs
  `SCRIPT_AUTOCONF_COMMAND`. Pass `PASSWORD` only when the install configured a
  root password. Override `LOGIN_PROMPT` when the guest uses a non-default
  login prompt.

- `script_change_floppy IMAGE`
  Swaps the first floppy to `IMAGE` and waits briefly for the change to settle.
  Use after the installer prompts for different media.

- `script_set_boot DISK`
  Sets the next boot device order (e.g. `a`, `c`).

### `serial.sh`

Serial-pipe install automation primitives. `serial_*` helpers read guest output
from the serial pipe and write answers back over the same channel.

Serial scripting prints a combined transcript of guest output and host input:
ordinary serial text received from the guest is prefixed with `➡️`, text
transmitted to the guest is prefixed with `⬅️`, and the line that satisfies the
active wait is prefixed with `✅`. Dialog adapter screens also use this channel.

Functions:

- `serial_wait [-l] [-r] TEXT [TEXT ...]`
  Waits until each `TEXT` appears in the serial stream, in argument order. Pass
  `-l` to match trimmed full lines, or `-r` to match extended regexes.

- `serial_send TEXT`
  Writes `TEXT` plus a newline to the guest serial pipe and logs it in the
  serial transcript.

- `serial_prompt [-r] QUESTION [QUESTION ...] ANSWER`
  Waits for one or more serial prompt lines, then sends `ANSWER` to the serial
  pipe.

- `serial_shell_start`
  Starts a serial-backed interactive shell and waits for its prompt. Use this
  when several phases must share one shell process.

- `serial_shell_send [--no-wait] COMMAND`
  Sends one command to an active serial shell and waits for the serial shell
  prompt to return unless `--no-wait` is passed.

- `serial_shell_exit`
  Sends `exit` to the active serial shell and waits for the screen prompt
  to return.

- `serial_shell [--no-wait] COMMAND [COMMAND ...]`
  Uses the keyboard/screen shell to start `sh -i` with stdin, stdout, and
  stderr redirected to the guest serial device, then sends each command over
  the serial pipe. Pass `--no-wait` for commands that take over, reboot, or
  prompt for later serial input.

### `fdisk.sh`

Host-side fdisk automation helpers. These build the standard swap/root
partition layout by starting fdisk from a serial shell and driving its
interactive prompts over the serial pipe.

Functions:

- `script_fdisk DEVICE SWAP_MB`
  Creates swap and root partitions, sizing swap with fdisk's `+sizeM`
  shorthand and giving root the remaining cylinders fdisk advertises.

## Slackware Tagfiles

### `slackware.sh`

Slackware-specific install media and tagfile preparation.

#### Tagfiles and tagsets

Slackware's installer reads a per-series *tagfile* that lists each package with a
state: `ADD` (install), `REC` (recommended), `OPT` (optional), or `SKP` (skip).
retrodist drives unattended installs by generating those tagfiles from `*.tag`
*tagset* files in the config directory. A tagset lists `series package state`
rules; its filename stem names the set. Three are conventional:

| Tagset | Purpose |
|---|---|
| `default.tag` | Slackware's own stock selection, generated by `retro tagfile` — every package with the flag and description it ships with on the install media. |
| `full.tag` | Install everything: a `series * ADD` wildcard per series. |
| `base.tag` | Minimal base system: the `a` series with the IDE kernel forced on; other series fall through to the default `SKP`. |

A rule is either a series wildcard (`series * STATE`) or a specific override
(`series package STATE`). At install time they are applied so wildcards land
first and specific overrides win, so order within a file does not matter. A
tagset in the variant directory shadows a same-named one in the version
directory above it. `INSTALL_TAGSETS` (default: `full`) is the space-separated
list of stems that take effect.

#### Generating `default.tag` — `retro tagfile`

- `retro_tagfile`
  Top-level handler for `retro tagfile CONFIG`. Extracts the install media, then
  writes `default.tag` to the config directory from the media's own tagfiles.

- `slackware_extract_tagfiles TAGROOT` / `slackware_extract_tagfiles_from_iso ISO TAGROOT`
  Extract each series' `tagfile` and description files (`disk*`, `*.txt`) into
  `TAGROOT` (the gitignored `tagfile.d/`), from staged FAT packages or an ISO.
  `slackware_consolidate_tagfiles SRCROOT TAGROOT` gathers every disk of a series
  into its first-disk directory (`TAGROOT/<series>1`).

- `slackware_generate_default_tag TAGROOT OUTFILE`
  Writes `default.tag` from the extracted sources: a `series * SKP` baseline per
  series, then every package with its shipped state and a description comment.

#### Applying tagsets at install time

- `slackware_prepare_tagfiles`
  Called by `autoinst_prep`. Enumerates the package universe (in shell, since it
  needs `find`/`7z`) from staged FAT media or the install ISO into a
  `target<TAB>series<TAB>pkg` stream, then a single awk pass writes every
  `tagfile`/`tagfile.new` with each package's final state: default `SKP`,
  overridden by the series' `*.tag` wildcard and then specific-package rules.
  ISO-derived custom PATH tagfiles are written under each source disk
  directory (e.g. `fat/tagfiles/a3/tagfile`) because the color `cpkgtool`
  looks up tagfiles by the current disk directory name.
  Records the series with any selected package in `fat/disksets.txt`, which the
  in-guest installer reads to know which sets to install.

- `slackware_universe_from_staged PKGROOT` / `slackware_universe_from_iso`
  Emit the `target<TAB>series<TAB>pkg` universe (package names version-stripped)
  for staged FAT media and for an ISO listing (`SLACKWARE_PKGLIST`) respectively.

- `slackware_collect_tagsets`
  Collects `*.tag` rule files (variant shadows version directory) into
  `SLACKWARE_TAGSETS`, filtered to the stems in `INSTALL_TAGSETS`.

- `slackware_staged_pkg_root` / `slackware_ensure_pkglist ISO`
  Locate the staged package root on FAT media, and list an ISO's packages once
  into `SLACKWARE_PKGLIST` (cached for the invocation).
