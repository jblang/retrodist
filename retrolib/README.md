# retrolib

This directory contains the host-side shell library sourced by the `retro`
entry point at startup. Every file here is sourced unconditionally, so all
functions are available regardless of the command being run.

## Compatibility Notes

### Shell

These scripts must work on GNU bash 3.2.57, the version shipped with macOS.
Avoid features added in bash 4.0 or later:

- No `mapfile` or `readarray` ‚Äî iterate with `while IFS= read -r`.
- No associative arrays (`declare -A`).
- No `|&` (pipe stderr to next stage).
- No `wait -n`.

`[[ ]]`, `$(( ))` arithmetic, C-style `for ((‚Ä¶))` loops, arrays, `read -ra`,
`<<<` here-strings, and `printf -v` are all available in 3.2.

### External commands

External commands must use only POSIX-specified flags and behavior. The scripts
run on macOS (BSD userland) and Linux (GNU coreutils), so avoid flags that work
on one but not the other:

- `grep`: `-q`, `-F`, and `-E` are POSIX. Avoid `-P` (Perl regex, GNU only).
- `sed`: avoid `-i` for in-place editing ‚Äî GNU requires no argument; BSD
  requires an explicit empty-string argument. BRE patterns and `-E` for ERE
  are POSIX.
- `find`: `-type`, `-name`, `-print`, `-exec`, and `-prune` are POSIX.
  `-maxdepth`, `-mindepth`, and `-iname` are extensions (XSI/BSD/GNU) not in
  base POSIX.
- `sort`: `-u` and `-o` are POSIX. Avoid `-V` (version sort, GNU only).
- `awk`: standard POSIX awk only. Avoid gawk-specific built-ins such as
  `gensub`.
- `ls`: `-l` and `-a` are POSIX. Avoid `-A` (hides `.`/`..` but still shows
  dotfiles) and `-h` (human-readable sizes).
- `ps`: `-p PID` is POSIX; the `-o` format specifier is an XSI extension.
- `truncate`: not in POSIX. Prefer `dd` or `perl` when file padding is needed.

## `helpers.sh`

Shared utilities used across all other modules.

- `retro_config_file [DIR] FILE`
  Searches `DIR` (default: `$CONFDIR`) then its parent for `FILE`. Prints the
  path and returns 0 when found; returns 1 when not found. Config files can be
  shared at the version level by placing them one directory above the variant.

- `retro_prereq`
  Detects the host package manager (Homebrew, apt-get, dnf, pacman, MSYS2
  pacman) and installs the required tools: QEMU, 7z, unzip, wget, bchunk,
  xorriso, lsof, jq, and ssh. Supports `--dry-run`.

- `autoinst_prep`
  Stages the `autoinst/` tree onto `qemu.d/fat/` for the current config. Copies
  `autoinst.sh` as `fat/autoinst`, copies the whole tree to `fat/autoinst.d/`,
  then copies the distro's `autoinst.sh` and `autoconf.sh` manifests to
  `fat/autoinst.d/distro/`. See [autoinst/README.md](../autoinst/README.md).

## Config Directory Hierarchy

Distro configs live at `distro/version/variant/` (e.g., `slackware/3.0/walnut/`).
`retro_config_file NAME` searches `$CONFDIR` (the variant directory) first, then
its parent, so files can be shared at the version level.

All per-config files are optional:

| File | Purpose |
|---|---|
| `config.sh` | Distro-wide settings; loaded before QEMU config |
| `qemu.sh` | Set `QEMU_PROFILE`, RAM, disk size, net device, extra QEMU args |
| `extract.sh` | Set `EXTRACT_*` vars and call `extract_install_files` |
| `script.sh` | Host-side install automation; sourced after QEMU starts |
| `autoinst.sh` | In-guest install manifest; copied to `fat/autoinst.d/distro/` |
| `autoconf.sh` | In-guest first-boot manifest; copied to `fat/autoinst.d/distro/` |
| `download.txt` | `filename url` pairs for wget |
| `slackmirror.txt` | Slackware version for official mirror download |
| `debmirror.txt` | Debian release name for archive.debian.org download |
| `download.sh` | Custom download logic |
| `cdrom.txt` | Reference to a `cdrom/` config; ISOs are linked into `qemu.d/` |
| `pkgskip.txt` | Slackware packages to mark `SKP` in generated tagfiles |

## `qemu.d/` ‚Äî Staging Area

Generated by `retro extract` for each distro config. Contains:

- `hda.img` ‚Äî primary hard disk (qcow2, created when startup media is present)
- `boot.img` ‚Üí symlink to the boot floppy image
- `root.img` ‚Üí symlink to the root floppy image
- `install.iso` ‚Üí symlink to the install ISO
- `fda.img`, `hdc.iso`, etc. ‚Äî optional additional media
- `fat/` ‚Äî exposed to the guest as a writable FAT disk at `/dev/hdb1`
- `fat/autoinst` ‚Äî in-guest install runner (copied from `autoinst/autoinst.sh`)
- `fat/autoinst.d/` ‚Äî shared runtime and distro manifests (see [autoinst/README.md](../autoinst/README.md))
- `qmp.sock` ‚Äî QMP Unix socket (live while QEMU is running)
- `ttyS0.sock`‚Ä¶`ttyS3.sock`, `lp0.sock` ‚Äî serial and parallel port sockets

## `download.sh`

Asset download helpers and top-level `retro download` handler.

- `download_list MANIFEST DESTDIR`
  Downloads `filename url` pairs from a manifest file using wget, skipping
  files that already exist.

- `download_files URL_BASE DESTDIR FILE‚Ä¶`
  Downloads named files from a base URL.

- `download_directories URL_BASE DESTDIR DIR‚Ä¶`
  Recursively downloads named directories from a base URL using wget
  `--recursive`, computing `--cut-dirs` automatically.

- `download_slackware VERSION`
  Downloads a full Slackware release tree from mirrors.slackware.com.

- `download_debian RELEASE`
  Downloads a Debian release from archive.debian.org. Handles per-release
  layout differences (Debian 0.93R6 through post-potato).

- `download_config_assets CONFIG_DIR DESTDIR`
  Dispatches all configured download mechanisms for a config directory:
  `download.txt`, `slackmirror.txt`, `debmirror.txt`, and `download.sh`.

- `download_cdrom_assets CONFIG_DIR` / `link_cdrom_isos CONFIG_DIR DESTDIR`
  Downloads assets for a `cdrom/` config referenced by `cdrom.txt` and links
  resulting ISO files into the distro extraction directory.

- `retro_download`
  Top-level handler for `retro download CONFIG`. Calls `download_all` against
  `$CONFDIR` and `$ORIGDIR`.

## `extract.sh`

Media extraction and staging helpers and the top-level `retro extract` handler.

Extraction is controlled by setting `EXTRACT_*` variables before calling
`extract_install_files`. The function detects whether the source is an archive
or a directory and dispatches accordingly.

`EXTRACT_*` variables:

- `EXTRACT_SOURCE` ‚Äî archive file or source directory. Relative paths are
  resolved under `$ORIGDIR`.
- `EXTRACT_BOOT_IMAGE` ‚Äî boot floppy image name to extract.
- `EXTRACT_ROOT_IMAGE` ‚Äî root floppy image name to extract.
- `EXTRACT_EXTRA_IMAGES` ‚Äî array of additional image names to extract.
- `EXTRACT_FAT_FILES` ‚Äî array of files to extract into `fat/`.
- `EXTRACT_PACKAGES` ‚Äî directory path inside the archive to extract as
  `fat/packages/`. Use `"."` to extract the whole archive.

Key functions:

- `extract_install_files`
  Extracts or copies the configured images and packages, links `boot.img` and
  `root.img`, and resets `EXTRACT_*` afterwards.

- `debian_extract_fat_image IMAGE DEST FILE‚Ä¶`
  Extracts selected files from a FAT image using 7z and lowercases filenames.

- `retro_link_boot_root BOOT ROOT`
  Updates the `boot.img` and `root.img` symlinks in `qemu.d/`.

- `retro_extract`
  Top-level handler for `retro extract CONFIG`. Runs `retro_download`, sources
  the distro's `extract.sh`, and calls `autoinst_prep` when the distro has an
  `autoinst.sh` manifest. Re-runs `autoinst_prep` on subsequent calls to pick
  up manifest changes without a full re-extract.

## `qemu.sh`

QEMU configuration, port allocation, drive building, and top-level boot,
install, reset, and package handlers.

### Configuration variables

All `QEMU_*` variables can be set in the distro's `qemu.sh` or via the
environment. `load_qemu_config` applies defaults, sources the distro's
`qemu.sh`, resolves the profile, assigns ports, and finalizes network config.

System:

- `QEMU_SYSTEM` ‚Äî QEMU binary. Default: `qemu-system-i386`.
- `QEMU_PROFILE` ‚Äî era profile name. Default: `default`. See [Profiles](#profiles).
- `QEMU_MACHINE` ‚Äî `-machine` argument. Set by profile.
- `QEMU_RAM` ‚Äî RAM size. Set by profile.
- `QEMU_SMP` ‚Äî CPU count. Default: `1`.
- `QEMU_ACCEL` ‚Äî acceleration flags. Default: `-accel tcg`.
- `QEMU_EXTRA` ‚Äî extra QEMU arguments appended verbatim.
- `QEMU_DISPLAY` ‚Äî display backend flags. Default: `-display cocoa` on macOS,
  `-display gtk` elsewhere.

Storage:

- `QEMU_HD_FORMAT` ‚Äî disk image format. Default: `qcow2`.
- `QEMU_HD_SIZE` ‚Äî primary disk size. Set by profile.
- `QEMU_HD_CREATE_OPTIONS` ‚Äî extra `-o` options for `qemu-img create`.
- `QEMU_HDA_OPTIONS` ‚Äî extra options appended to the `hda` drive string.
- `QEMU_FDTYPE_A` / `QEMU_FDTYPE_B` ‚Äî floppy type globals. Default: `144`.
- `QEMU_BOOT_ORDER` ‚Äî `-boot order=` argument. Auto-set at install time.

Network:

- `QEMU_NET_TYPE` ‚Äî `user` (default), `jump`, or `none`. See [Network modes](#network-modes).
- `QEMU_NET_DEVICE` ‚Äî NIC model. Set by profile.
- `QEMU_INTERNET` ‚Äî raw `-netdev`/`-device` string for internet access.
  Overrides `QEMU_NET_TYPE` when set.
- `QEMU_RETRONET` ‚Äî raw `-netdev`/`-device` string for jump-box access.
  Overrides `QEMU_NET_TYPE` when set.

Ports:

- `QEMU_SSH_BASE` / `QEMU_SSH_PORT` ‚Äî host port for guest :22. Default base: 2200.
- `QEMU_TELNET_BASE` / `QEMU_TELNET_PORT` ‚Äî host port for guest :23. Default base: 2300.
- `QEMU_HTTP_BASE` / `QEMU_HTTP_PORT` ‚Äî host port for guest :80. Default base: 8000.
- `QEMU_MONITOR_BASE` / `QEMU_MONITOR_PORT` ‚Äî telnet monitor port. Default base: 5555.
- `QEMU_QMP_SOCKET` ‚Äî QMP Unix socket name. Default: `qmp.sock`. Set `none` to disable.

Serial / parallel:

- `QEMU_SERIAL_SOCKET_COUNT` ‚Äî number of serial sockets. Default: `4`.
- `QEMU_SERIAL_SOCKET_PREFIX` ‚Äî socket name prefix. Default: `ttyS`.
- `QEMU_PARALLEL_SOCKET_COUNT` ‚Äî number of parallel sockets. Default: `1`.
- `QEMU_PARALLEL_SOCKET_PREFIX` ‚Äî socket name prefix. Default: `lp`.

Install scripting:

- `QEMU_INSTALL_SCRIPT` ‚Äî path to a `script.sh` sourced after QEMU starts.
  Auto-set from the distro's `script.sh` when running `retro install`.

### Profiles

Set `QEMU_PROFILE` in the distro's `qemu.sh` to select era-appropriate
hardware defaults. Any variable set before `qemu_apply_profile` is not
overwritten, so individual overrides still take effect.

| Profile | Machine | RAM | Disk | NIC |
|---|---|---|---|---|
| `default` | isapc | 16M | 500M | ne2k_isa |
| `linux-0.99` | isapc | 64M | 500M | ne2k_isa |
| `linux-1.0` | isapc | 64M | 512M | ne2k_isa |
| `linux-1.2` | isapc | 64M | 2G | ne2k_isa |
| `linux-2.0-isa` | isapc | 64M | 2G | ne2k_isa |
| `linux-2.0` | pc | 64M | 8G | tulip |
| `linux-2.2` | pc | 64M | 8G | tulip |
| `linux-2.4` | pc | 128M | 8G | tulip |

### Network modes

Set `QEMU_NET_TYPE` in the distro's `qemu.sh` or in the environment.

- `user` (default) ‚Äî SLIRP NAT with host port forwards: `QEMU_SSH_PORT`‚Üí:22,
  `QEMU_TELNET_PORT`‚Üí:23, `QEMU_HTTP_PORT`‚Üí:80.
- `jump` ‚Äî socket connection to the jump box on port 1234. Start `jump run`
  first. Enables FTP transfers from the host.
- `none` ‚Äî no network interface.

Setting `QEMU_INTERNET` or `QEMU_RETRONET` directly bypasses `QEMU_NET_TYPE`.

### Top-level handlers

- `retro_boot` ‚Äî `retro boot CONFIG [QEMU_ARGS‚Ä¶]`
  Extracts, builds drives, assigns ports, starts QEMU. For `retro install`,
  also sources the install script via `qemu_run_with_install_script`.

- `retro_package` ‚Äî `retro package CONFIG [--hda] [QEMU_ARGS‚Ä¶]`
  Calls `retro_boot`, writes `retro.sh` (POSIX sh) and `retro.bat` (Windows
  cmd) launchers into `qemu.d/`, and tars the directory.

- `retro_reset` ‚Äî `retro reset CONFIG`
  Prompts and then removes `qemu.d/` and extracted files for the config.

## `qmp.sh`

QMP protocol helpers. Used by `script.sh` during automated installs and
exposed through the `qmp` entry point for interactive use.

Connection variables:

- `QEMU_QMP_SOCKET` ‚Äî QMP socket name or path. Default: `qmp.sock`.
- `QMP_TIMEOUT` ‚Äî `nc` timeout in seconds. Default: `1`.

VGA dump variables:

- `VGA_ADDR` ‚Äî physical address of VGA text memory. Default: `0xb8000`.
- `VGA_COLS` / `VGA_ROWS` ‚Äî screen dimensions. Default: 80√ó25.
- `VGA_MEM_BYTES` ‚Äî bytes to read. Default: `32768`.

Key functions:

- `qmp_init` ‚Äî sets defaults, checks prerequisites (nc, jq, xxd), validates VGA config.
- `qmp_hmp_command COMMAND` ‚Äî sends an HMP command through QMP.
- `qmp_sendkey KEY` ‚Äî sends a QEMU sendkey token.
- `qmp_send_string TEXT` ‚Äî types a string into the guest character by character.
- `qmp_send_line TEXT` ‚Äî types a string and presses Return.
- `qmp_change_image IMAGE [DEVICE]` ‚Äî changes floppy media at runtime.
- `qmp_eject_disk [DEVICE]` ‚Äî ejects media from a block device.
- `qmp_boot_disk DEVICE` ‚Äî sets the next boot device order.
- `qmp_vga_dump_text` ‚Äî dumps and decodes VGA text memory as a plain-text screen.

## `script.sh`

QMP-driven install automation primitives. These are sourced when QEMU starts
during `retro install` and are the building blocks for per-distro `script.sh`
manifests.

Timing variables:

- `WAIT_TIMEOUT` ‚Äî default timeout in seconds for screen-match waits. Default: `60`.
- `WAIT_INTERVAL` ‚Äî polling interval in seconds. Default: `1`.

Functions:

- `script_wait_screen_text TEXT [TIMEOUT [INTERVAL]]`
  Polls VGA text memory until `TEXT` appears anywhere on screen.

- `script_wait_screen_line TEXT [TIMEOUT [INTERVAL]]`
  Polls until `TEXT` appears as a trimmed full line on screen.

- `script_boot_lilo [PROMPT]`
  Waits for a LILO boot prompt (default `boot:`) and presses Return.

- `script_answer_prompt PROMPT [ANSWER]`
  Waits for a full-line prompt and sends an answer (or just Return).

- `script_change_floppy PROMPT [IMAGE [ANSWER]]`
  Waits for a prompt, swaps the first floppy image, then sends an answer.

- `script_login PROMPT [USER]`
  Waits for a login prompt and types a username (default: `root`).

- `script_run_autoinst PROMPT`
  Waits for a shell prompt, then mounts `/dev/hdb1` at `/retro` and launches
  `autoinst`.

- `script_finish_reboot [DISK [PROMPT [TIMEOUT]]]`
  Waits for the autoinstall completion prompt, sets the boot disk, and confirms
  the reboot.

- `script_press_key KEY` / `script_send_return`
  Direct key-send wrappers.

## `slackware.sh`

Slackware-specific install media and tagfile preparation.

- `slackware_prepare_tagfiles`
  Called by `autoinst_prep`. Generates `tagfile` and `tagfile.new` files for
  every package series found on staged FAT media or inside the install ISO,
  marking all packages `ADD`. Applies any per-distro package skip rules from
  `pkgskip.txt`.

- `slackware_set_tagfile_package PKG STATE`
  Sets a package's state (`ADD`, `SKP`, `OPT`, etc.) in every staged tagfile.

- `slackware_build_tagfiles` ‚Äî builds tagfiles from staged `fat/packages/`.
- `slackware_build_tagfiles_from_iso ISO PATH_ROOT` ‚Äî builds tagfiles from
  package names listed inside an ISO without extracting them.
